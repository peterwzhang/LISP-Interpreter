Research/Learning

We took a group approach for the research required for this project. This involved meeting up as a group every Tuesday and Thursday from 12:30 p.m. CST to 6:30 p.m. CST. During these meetings, we discussed the goals of the project, the language to use, and how to impement each part of the interpreter (scanner, parser, and the interpreter. We popcorn read the Crafting Interpreters textbook in order to learn about the general grammar of languages. This was a great opportunity to get to learn about interpreter concepts together as a group and stay on the same page. After we learned about the general grammer of languages. We then learned about LISP, we played around with a Common Lisp by using SBCL. This helped us understand some general Lisp concepts. Then we read through Kamin's Pascal Lisp to see the flavour of Lisp we will be implementing. After all of this was complete we could then begin to work on our own implementation of the Kamin's Lisp.

Testing Script

We used a bash script to help automate our tests. The bash script is named "tester.sh" and is located in the main directory. It is hardcoded to specifically work with our project directory. We updated a 1994 C++ version of Kamin's Lisp Interpreter to work with a modern day C++ compiler. The testing script ask to run either a specified single test case or all of the test cases. The tester will then run the test case(s) on the C++ Kamin Interpreter and our own interpeter and compare the outputs. If they resulting outputs are the same then the test passes.
